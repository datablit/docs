# Getting Started with Event Ingestion

Welcome to DataBlit! This guide will walk you through the process of setting up event ingestion for your applications.

## Overview

Event ingestion is the process of collecting and sending user interaction data from your applications (Android, iOS, or web) to DataBlit for analysis. This data helps you understand user behavior and make data-driven decisions.

## Prerequisites

Before you begin, make sure you have:

- Access to your application's codebase
- Basic understanding of your development platform

## Step-by-Step Setup

### 1. Create Your DataBlit Account

First, you'll need to access the <a href={`${process.env.NEXT_PUBLIC_DATABLIT_CONSOLE_URL}`} className="underline font-medium" > DataBlit Console</a>

### 2. Create or Select an existing project

Once logged in, you'll need to work within a Project

- **New Project**: Create a new project to organize your data collection
- **Existing Project**: Select an existing project if you're adding another data source <br/>

### 3. Set Up Data Sources

A data source represents where your events are coming from. We recommend creating separate sources for different applications:

## What's Next?

After completing the setup:

1. **Test Your Integration**: Send test events to verify everything is working
2. **Configure Events**: Set up the specific events you want to track
3. **Explore Analytics**: Visit your dashboard to see your data in action
4. **Scale Up**: Add more sources and events as your needs grow

- Review the troubleshooting section in each SDK guide
